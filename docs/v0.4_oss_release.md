# v0.4 開源發布準備

## 目的

v0.4 的目標是讓 Memosyne 能安全公開，且不需要讀者理解任何私有 vault 脈絡。

對外應傳達的產品定位：

> 本地優先的 AI Agent 個人記憶基礎設施，提供 ingest、hybrid retrieval、graph memory、correction、evaluation 與 MCP integration。
文檔與示例資料規範遵守 workspace `AGENTS.md`。

## 公開定位

不要先列研究技巧清單。

應先說明問題：

- AI Agent 缺少長期脈絡。
- 長期脈絡散落在 journal、chat、notes、profile 中。
- 本地記憶層應可搜尋、可修正、可審計、可供 Agent 呼叫。

再解釋架構如何解決這些問題。

## 發布邊界

公開 repo 應包含：

- source code
- docs
- synthetic sample vault
- sample evaluation set
- install instructions
- MCP setup guide
- privacy model
- contribution guide
- license

公開 repo 不得包含：

- private `_vault` contents
- API keys
- 個人 benchmark reports
- private cache files
- 含真實值的 `.env`
- 由私有資料生成的 DB
- 除示例外的本機絕對路徑

## Sample Vault

sample vault 是必要項。沒有 sample data，外部使用者很難評估系統。

最低結構：

```text
sample_vault/
  10_Profile/
    bio.md
    career.md
    preferences.md
  20_AI_Chats/
    Synthetic/
      sample_chat.md
  30_Journal/
    2026/
      260101.md
      260107.md
  40_Projects/
    sample_project.md
  50_Knowledge/
    sample_note.md
```

sample 應展示：

- person/place/event extraction
- temporal search
- AI chat vs journal retrieval
- correction flow
- graph association query
- negative / abstention query

sample 內容必須合成。不要把真實私有資料做簡單匿名化後發布；若需要借用結構，必須重寫到不可回推。

## 開源文檔集合

公開版本需要：

- `README.md`：產品概覽、quickstart、架構、限制
- `docs/quickstart.md`：clean-machine setup

- `docs/evaluation.md`：Eternal Mirror 與 Augury

若目標是廣泛 OSS adoption，公開預設 README 建議使用英文；中文文檔可保留為並列版本。

## Packaging 目標

理想安裝路徑：

```bash
pip install memosyne
```

pre-release 可接受路徑：

```bash
git clone https://github.com/<owner>/memosyne
cd memosyne
pip install -e .
memosyne init --sample
```

Package metadata 應定義：

- Python version range
- console script entry point
- dependencies
- optional extras for local LLM, cloud LLM, eval, dev

建議 extras：

```text
memosyne[local]
memosyne[cloud]
memosyne[eval]
memosyne[dev]
```

## Privacy Requirements

`docs/privacy.md` 應回答：

- 哪些資料只留在本地？
- 哪些命令可能呼叫 cloud LLM？
- 哪些命令完全不需要 LLM？
- 哪些檔案是由私有資料衍生？
- 哪些路徑必須 gitignore？
- dry-run correction tools 如何運作？
- 使用者如何刪除或重建衍生索引？

預設行為應偏向本地；cloud calls 必須明確 opt-in。

## Evaluation Story

Memosyne 的差異化之一是內建 retrieval evaluation。

公開文檔應說明兩種評估：

- Eternal Mirror：用生成的 HyQE questions 做 self-supervised round-trip retrieval。
- Augury：用人工撰寫的 golden questions 測真實使用意圖。

sample vault 應附一組極小 evaluation set，讓 contributor 可執行：

```bash
memosyne eval --sample
```

## Security And Cleanup Checklist

公開前：

- rotate 曾經進入 repo 的 key
- 必要時從 history 移除 private key files
- 確認 `.gitignore` 包含 private vault 與 generated DBs
- 跑 secret scanner
- 確認 sample data 完全合成
- 確認 submodule 不指向 private memory content
- 確認 Docker image 不會 copy private files
- 移除公開文檔中的本機絕對路徑，除非是明確示例
- 搜尋並移除私有查詢、真實人名、私人關係線索與私有事件描述

## 開源範圍控制

避免承諾：

- hosted sync
- multi-user collaboration
- mobile apps
- perfect memory extraction
- medical/legal-grade correctness
- universal agent behavior

明確限制：

- retrieval quality 依賴來源資料與 enrichment 品質
- graph memory 可能放大 extraction errors
- local model setup 可能較重
- evaluation metrics 不能取代人工 review

## 出口標準

v0.4 完成條件：

- clean install works
- sample vault demo works
- public docs 不需要任何私有脈絡即可理解
- privacy boundary 明確
- 不包含 private data 或 keys
- 至少一條 MCP client setup path 已驗證
- evaluation 可在 sample data 上執行

到這個階段，Memosyne 才適合被宣傳為開源個人記憶基礎設施，而不是內部研究 repo。
