# v0.3 使用產品化

## 目的

v0.3 的目標是讓 Memosyne 能被穩定地日常操作。

目前系統能力完整，但入口偏腳本化。產品化不是新增功能，而是建立單一清楚的 command surface、明確前置條件、可預期維護流程與可理解的失敗訊息。
文檔與示例資料規範遵守 workspace `AGENTS.md`。

## 產品目標

技術型使用者應能在不先閱讀內部實作的情況下完成：

1. 建立 vault
2. 入庫檔案
3. 重建索引
4. 搜尋記憶
5. 檢查健康狀態
6. 啟動 MCP server
7. 安全修正記憶

## 建議 Command Surface

目標 CLI：

```bash
memosyne init
memosyne ingest
memosyne search "測試查詢"
memosyne rebuild
memosyne health
memosyne mcp
memosyne correct ...
memosyne slumber --stats
```

第一版可以只是包裝既有腳本，不需要立即大重構。重點是先穩定公開介面。

## Scope

允許：

- package entry point
- CLI wrappers
- health checks
- 更清楚的錯誤訊息
- 環境驗證
- 文檔整理

延後：

- 新 retrieval algorithm

## Operational Health Checks

`memosyne health` 應檢查：

- Python 版本是否支援
- 依賴套件是否可 import
- vault path 是否存在
- Chroma DB 是否存在或可建立
- BM25 index 是否存在
- Tapestry DB 是否存在
- Chronicle DB 是否存在
- embedding model 是否可從 cache 載入，或是否需要網路
- enrichment 所需 LLM provider 是否可用
- MCP server 是否可正常 import
- secret 是否誤放在公開路徑

輸出應能直接指示下一步：

```text
[ok] Python 3.11
[ok] Kuzu import
[warn] embedding model not cached; first search may need network access
[fail] missing Personal_Brain_DB/_vault
```

## Error Handling 要求

常見失敗應說明下一步，而不是只丟 raw traceback：

- virtualenv 未啟用
- Python 版本錯誤
- 缺少 `kuzu`
- Hugging Face model 未 cache 且無網路
- Ollama 未啟動
- cloud LLM key 缺失
- Aletheia 修正後索引過期
- vault submodule dirty

## Configuration Model

目標 config：

```toml
[paths]
vault = "Personal_Brain_DB/_vault"
spring = "spring"
system = "Personal_Brain_DB/00_System"

[models]
embedding = "paraphrase-multilingual-MiniLM-L12-v2"
enrichment = "gemma4:26b"
aggregation = "proxy:claude-opus-4-6"

[retrieval]
default_top_k = 5
auto_route = true
muse_mode = "soft"
```

v0.3 可以先保留程式內預設值，但文檔要說清楚配置方向。

## MCP 產品化

MCP server 應穩定暴露：

- `search_memory`
- `read_file`
- `get_profile`
- `list_journals`
- `get_memory_health`
- `optimize_memory`
- `query_memory_at_time`
- `get_entity_timeline`
- Aletheia correction tools
- `memosyne_guide`

v0.3 至少應標準化以下行為：

- write/correction tools 預設 dry-run
- 每個 retrieval result 都包含 path
- 錯誤訊息清楚
- 不做 silent writes

完整 schema 統一可延後，但公開行為不應頻繁變動。

## 驗證

v0.3 release 前：

```bash
workon personal-memory
memosyne health
memosyne search "測試查詢"
memosyne slumber --stats
memosyne mcp --check
```

若 `memosyne` 尚未實作，需先以既有命令驗證並在文檔中標示：

```bash
python Personal_Brain_DB/00_System/vectorize.py --query "測試查詢" --top 3
python Personal_Brain_DB/00_System/slumber.py --stats
python Personal_Brain_DB/00_System/mcp_server.py
```
